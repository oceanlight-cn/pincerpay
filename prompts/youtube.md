# YouTube Channel Rules

## Tone
Substantive, educational, developer-focused. Every video should teach something concrete. No fluff intros, no "smash that like button."

## Content Types

### Walkthrough Videos (primary)
- Screen recording of building something with PincerPay
- Start with the finished result, then show how to get there
- Code should be visible and readable (large font, dark theme)

### Deep Dive Explainers
- Architecture diagrams, protocol flows, technical comparisons
- Use animations/diagrams sparingly but effectively
- Every claim backed by code or data

## SEO Keywords
Include these naturally: "ai agent payments", "x402 protocol", "usdc payments", "solana payments", "api monetization", "micropayments", "machine to machine payments"

## Anti-Patterns
- No "Hey guys, welcome back to the channel!"
- No asking for likes/subscribes in the description (only at the end of the video itself)
- No clickbait titles: "This CHANGES EVERYTHING" or "You WON'T BELIEVE..."
- No filler: "Before we get started, let me tell you about..."
